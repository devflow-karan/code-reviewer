# GitHub Integration & Setup Guide

This guide describes how to set up, configure, and use the AI Code Review Agent with GitHub.

---

## 🏗️ Overview

The AI Code Review Agent supports GitHub Pull Requests (PRs). It processes code changes asynchronously using a producer-consumer architecture (Express + BullMQ + Redis) and reviews them using the Google Gemini API to enforce team coding standards.

```
 GitHub PR Event ──> Webhook Router ──> BullMQ (Redis) ──> Review Worker ──> Gemini AI ──> Post PR Review/Status
```

---

## ⚙️ Configuration & Environment Variables

To run the agent with GitHub, configure the following environment variables in your `.env` file:

```env
# Server Port
PORT=3000

# Redis connection details for BullMQ
REDIS_HOST=127.0.0.1
REDIS_PORT=6379

# GitHub Integration Config
GITHUB_TOKEN=ghp_your_personal_access_token # GitHub Personal Access Token (PAT) or GitHub App Installation Token
GITHUB_WEBHOOK_SECRET=your_github_webhook_sec # Secret token to verify GitHub webhook signatures

# Gemini AI config
GEMINI_API_KEY=your_gemini_api_key
GEMINI_MODEL=gemini-2.5-flash

# Review Rules Configuration
RULES_PATH=rules/rules.md                # Path to markdown file containing review guidelines
```

> [!NOTE]
> For production environments, it is recommended to use a **GitHub App** to handle authentication. In this case, `GITHUB_TOKEN` will be dynamically generated by the agent using the App's Private Key and App ID.

## 🔗 GitHub Webhook Configuration

To connect your GitHub repository to the AI agent:

1. Go to your GitHub Repository -> **Settings** -> **Webhooks**.
2. Click **Add webhook**.
3. Configure the webhook details:
   - **Payload URL**: `http://<your-agent-domain>/github/webhook`
   - **Content type**: `application/json`
   - **Secret**: Value of `GITHUB_WEBHOOK_SECRET`
   - **Which events would you like to trigger this webhook?**: Select **Let me select individual events**.
     - Check **Pull requests** (triggers on open, synchronize, close, reopen).
     - Check **Pull request review comments** (triggers when developers reply to inline comments).
4. Click **Add webhook**.

## 🚀 Core Features & Workflows

### 1. Automated Review on PR Lifecycle
- **Trigger**: Triggers instantly on PR creation (`opened`), code updates (`synchronize`), and reopenings (`reopened`).
- **Processing**: Enqueues a review job in BullMQ to respond to GitHub instantly, avoiding the 10-second webhook timeout.
- **Commit Status**: Sets commit status checks (`ai-code-review`) on the PR commit SHA. Updates to `success` or `failure` based on the review.
- **Auto-cancellation**: If a PR is `closed` or `merged`, the agent automatically cancels the pending review job and sets the commit status to `error`/`failure` or removes/cancels the check.

### 2. Conventional Commit & PR Metadata Check
- Validates the PR title and description against Conventional Commits.
- Validates commit messages for all commits included in the PR.
- Posts a review comment outlining metadata violations.

### 3. File Diff Filtering & Hashing
- Skips files inside `docs/` or `.agent/` folders.
- Skips binary or empty files.
- **Diff Hashing**: Hashes the diff of each reviewed file and embeds it in a hidden comment metadata tag. If a file is unchanged on subsequent updates, the agent skips re-reviewing it.

### 4. Incremental Review
- Keeps track of the last reviewed commit SHA for each file.
- On subsequent updates, queries the GitHub Compare Commits API to retrieve only the new diff (comparing last reviewed SHA with the current head SHA).
- Only reviews the new changes, reducing AI token usage.

### 5. Line-by-Line Live Reviews
- Posts code feedback as inline comments aligned directly to the updated line number using GitHub's Pull Request Reviews API.
- Combines comments into a single cohesive Review (using `pending` state and then submitting the review) to avoid individual comment notifications.
- Automatically falls back to a global PR comment if the line number is not part of the PR diff (e.g. context changes).

### 6. Interactive AI Re-Review & Thread Resolution
- **Developer Reply**: When a developer replies to an AI-generated review comment, the webhook triggers a `pull_request_review_comment` event.
- **Trigger Words**: If the reply contains keywords like `fixed`, `resolved`, `done`, `please recheck`, or `updated`, the agent enqueues a new review job to recheck the changes.
- **AI Conversation Handler**: Run asynchronously, it sends the discussion history to Gemini AI. If Gemini decides the issue is resolved, it automatically resolves/closes the conversation thread on GitHub using GitHub's GraphQL API (`resolveReviewThread` mutation).
